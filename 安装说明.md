# 鸟图智慧仓储（Birdy）安装与分发说明

版本信息以项目根目录 `**version-info.json**` 中的 `version` 为准。

## 一、环境要求


| 项目     | 建议                                      |
| ------ | --------------------------------------- |
| 操作系统   | Windows 10/11、macOS、Linux               |
| Python | **3.8～3.12**（3.13 可能可用，请以实际测试为准）        |
| 内存     | ≥ 8 GB（处理大批量连拍时更宽裕）                     |
| 磁盘     | ≥ 3 GB（含依赖与 `models/` 下权重；模型未随仓库时需自行放置） |
| GPU    | 可选；有 NVIDIA GPU 时可安装 CUDA 版 PyTorch 加速  |


## 二、获取代码与模型

1. 解压或克隆本仓库到本地，**工作目录应为包含 `src/`、`requirements.txt` 的根目录**。
2. **模型文件**（体积较大，常需单独拷贝或从网盘分发）应位于 `**models/`**（与 `src/` 同级），至少包括：
  - `bird-seg.pt` — 鸟体检测  
  - `bird-iden.pth`、`bird_info.json` — 本地物种分类（权重与索引表需配套）  
  - `birdeye.pt` — 鸟眼检测（若启用鸟眼功能）
3. `data/species/bird_species_list.csv` 为中英名称与学名对照清单（由 `bird_classification.json` 整理），用于名称差异时的统一规范与检索；建议保留随项目分发。
4. 若出现「zip / central directory」类报错，多为权重损坏或 Git LFS 未拉全，请核对文件大小后重新拷贝。

## 三、安装依赖

在**项目根目录**打开终端：

```bash
python -m venv .venv
```

**Windows：**

```bat
.venv\Scripts\activate
pip install -U pip
pip install -r requirements.txt
```

**macOS / Linux：**

```bash
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

**GPU 用户**：可到 [PyTorch 官网](https://pytorch.org) 选择 CUDA 版本安装 `torch`、`torchvision`，再执行：

```bash
pip install -r requirements.txt --no-deps
pip install opencv-python Pillow ultralytics numpy PyQt5 piexif geopy requests tqdm
```

（或先注释掉 `requirements.txt` 中的 `torch` / `torchvision` 两行后只装其余包。）

## 四、配置文件（可选但推荐）


| 文件                               | 作用                                                                                                    |
| -------------------------------- | ----------------------------------------------------------------------------------------------------- |
| `**src/doubao_api_config.json**` | 火山方舟豆包视觉 API：`api_key`、`api_base`、`models` 多模型轮换、`daily_token_limit_per_model`、`token_switch_ratio` 等 |
| `**src/amap_api_config.json**`   | 高德 Web 服务 Key；地名转坐标（GUI「高德API → 打开配置文件」）                                                              |
| `**src/geocoding_config.py**`    | 地理编码回退与其它地图开关（JSON 未填 Key 时可在此填写）                                                                     |
| `**src/gui_config.json**`        | GUI 保存的界面参数（运行后自动生成/更新）                                                                               |


将仓库分享给他人时，**请勿**把真实 API Key 一并公开。根目录 `**.gitignore`** 已忽略 `doubao_api_config.json`、`amap_api_config.json` 等；克隆后若缺少这些文件，首次打开 GUI 中的「豆包 / 高德」配置或运行相关流程时，程序会在 `src/` 下**自动生成**空密钥的 JSON 模板（见 `src/api_config_defaults.py`）。

## 五、启动方式

**务必在项目根目录执行**，或直接使用启动脚本（脚本会 `cd` 到 `src` 再运行）。

### Windows

双击 `**start_gui.bat`**，或在根目录执行：

```bat
python src\birdy_gui.py
```

### macOS / Linux

```bash
chmod +x start_gui.sh
./start_gui.sh
```

或：

```bash
python src/birdy_gui.py
```

### 命令行（批处理）

```bash
python src/birdy_cli.py -i ./你的图片文件夹 -o ./outputs
python src/birdy_cli.py --help
```

## 六、首次运行建议流程

1. 选择**图片文件夹**与**输出文件夹**。
2. 若使用连拍筛选：勾选「连拍检测」；保留结果会写入 `**{输出目录}/Screened_images/`**。
3. 物种识别可选**本地模型**或**豆包**（需在 `doubao_api_config.json` 配置方舟 Endpoint）。
4. 需要地名写入 GPS 时，配置高德或 `geocoding_config.py` 中的 Key。

## 九、问题排查


| 现象                      | 处理方向                                                    |
| ----------------------- | ------------------------------------------------------- |
| `No module named PyQt5` | 在已激活的 venv 中执行 `pip install -r requirements.txt`        |
| 找不到模型                   | 确认 `models/` 下 `.pt` / `.pth` 路径与大小正常                   |
| 豆包 404                  | 检查 `api_base` 与控制台 **推理接入点 ID**（`model` / `models`）是否一致 |
| 连拍后物种步骤无图               | 未勾选连拍时，需先有 `输出目录/Screened_images` 中的筛选图                 |


更多技术细节见 `**README.md**`。
